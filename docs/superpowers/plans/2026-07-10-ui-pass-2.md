# UI Pass 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish the interaction layer: viewer marker colors and brush cursor (jvo, gso), a tool palette (7tq), a visual empty state with drag-and-drop open (y59, 4tk), toolbar icons (6jq), and shortcut discoverability via a `?` overlay plus a native File menu (n5n).

**Architecture:** Marker colors move out of the GL shaders' baked constants into per-call parameters so defect markers and the brush cursor can differ; a new `--detect` teal replaces red for defect indication (red fights both film content and the operator's request). The palette grows out of the existing zoom cluster. Drag-drop uses Tauri 2's webview drag-drop events with a tiny backend path classifier. The shortcuts overlay renders the real key inventory; the native menu stays minimal (File: open actions) and reuses existing frontend flows via events.

**Tech Stack:** Rust (Tauri 2), Svelte 5 runes + TypeScript, WebGL (renderer.ts), vitest, cargo test.

## New color scheme (Task 1 fixes the values in one place)

| Meaning | Today | After |
|---|---|---|
| Detection tint (shader) | red `vec4(1.0, 0.05, 0.05, 1.0)` | teal `vec4(0.25, 0.82, 0.77, 1.0)` |
| Defect rings | red (baked) | teal `[0.25, 0.82, 0.77, a]` (param) |
| Brush cursor ring | red (baked, same shader) | white `[1.0, 1.0, 1.0, 0.9]` (param) |
| Paint strokes | amber `[1.0, 0.72, 0.24, 0.35]` | unchanged |
| Erase strokes | blue `[0.42, 0.69, 1.0, 0.3]` | neutral `[0.91, 0.90, 0.89, 0.35]` (text-1 at 35% — "lifting off", not a third hue) |
| CSS token | — | `--detect: #40d1c4;` in app.css with comment `/* defect markers; GL mirrors in renderer.ts/Viewer.svelte -- keep in sync. Non-text: 6.2:1 on --bg-0, 3.4:1 on --surround. */` |

Rationale recorded for the gate: teal is absent from film content (no skin-tone/red-cast collisions on colour scans, high visibility on B&W), amber stays the "operator marked this" colour, the cursor is neutral white so it reads over any tint.

fn path_kind(path: String) -> Result<String, String> {

## Definition of done

- The palette exposes brush/overlay/zoom controls with pressed states and keyboard parity.
- A first-run window explains itself and accepts drops; both open paths share code with the pickers.
- Every button has an icon; every shortcut is discoverable via ? and the File menu carries the two open actions with standard accelerators.
- NOT here: split/wipe compare (fuj), custom tooltip system (titles + overlay suffice for now), single-image export changes, detect progress (2vf).
