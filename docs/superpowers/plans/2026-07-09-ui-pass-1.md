# UI Pass 1: Design System and Core Chrome Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give TheUnduster its visual identity — the Darkroom character (graphite chrome, safelight-amber accent, neutral grey around images) — through a token-based design system, and rebuild the four pieces of core chrome on it: toolbar, status bar, notifications, filmstrip.

**Architecture:** All visual decisions live in CSS custom properties in `app.css` (palette, spacing, type, radii, motion), with component classes (`.btn`, `.toolbar-group`, `.badge`, slider restyle) built on them — later beads and pass 2 consume tokens, never raw values. The header becomes a grouped toolbar; the status text becomes a bottom-docked, three-zone status bar with a single prioritized activity slot; the shared `error` string becomes a toast stack plus a capped activity log; the filmstrip gets designed states and a coherent badge system. Frontend-only: no Rust changes, no behavior changes beyond where notifications surface.

**Tech Stack:** Existing stack. No component libraries — the app has ~8 controls; hand-rolled on tokens beats a dependency.

## Global Constraints

- Trunk-based: commit directly to `main`, atomic commits, tests green first. No emoji anywhere, including UI copy and badge glyphs (styled text characters like a check mark U+2713 are fine; emoji code points are not).
- WCAG 2.2 AA floor: every text/background pair at or above 4.5:1 (the token task lists computed ratios; keep them true if adjusting); visible focus states on every interactive element (`:focus-visible` ring token); hit targets at least 24px.
- `prefers-reduced-motion: reduce` disables all transitions/animations globally (one media block in app.css).
- The viewer canvas surround becomes the neutral grey token — chrome stays darker; nothing tints the area around film.
- The smoothness contract stands: no layout-shifting status text (the activity slot is fixed-height; zone widths stable), no animation on the pan/zoom path.
- Keyboard shortcuts remain authoritative; every toolbar button's `title` teaches its key.
- Existing tests stay green: vitest 30, svelte-check 0 errors (the 4-warning baseline may CHANGE if warnings' line numbers shift — same four kinds, count must not grow).

### Task 4 (6ic): Toasts and the activity log

**Files:**
- Create: `app/src/lib/toasts.ts` (store + pure helpers, unit-tested)
- Create: `app/src/lib/Toasts.svelte` (the stack)
- Modify: `app/src/App.svelte` (error sites migrate; log panel toggle in the status bar's right zone)

**Interfaces:**
- `toasts.ts` (pure, tested — the store is a plain module-level `$state` via Svelte 5 `.svelte.ts`... KEEP IT SIMPLER: a plain TS module with types + pure list helpers, the reactive array lives in App and is passed down; testable helpers):

```ts
export interface Toast {
  id: number;
  level: "info" | "error";
  message: string;
  count: number; // collapsed repeats
}

/** Append or collapse into the last toast when level+message match. */
export function pushToast(list: Toast[], level: Toast["level"], message: string, id: number): Toast[];
/** Remove by id. */
export function dismissToast(list: Toast[], id: number): Toast[];
/** Cap helper for the activity log. */
export function pushLog<T>(list: T[], entry: T, cap: number): T[];
```

Unit tests: push appends; identical consecutive push collapses (count increments, same id); different message appends; dismiss removes; pushLog caps at N dropping oldest.
- `Toasts.svelte`: fixed stack top-right (below the toolbar), each toast `--bg-2` card with `--err`/`--info` left border (4px), message in `--text-1`, dismiss button (a styled multiplication sign U+00D7, aria-label "Dismiss") on error toasts; info toasts auto-dismiss after 4s (timer per toast, cleared on unmount); `role="status"` for info, `role="alert"` for error; no entrance animation beyond a fast opacity transition (killed by reduced-motion).
- App: `toastList: Toast[]` + `activityLog: { time: string; level: string; message: string }[]` (cap 100). EVERY `error = ...` assignment becomes `pushToast(..., "error", ...)` + log append; the `error` state variable and its render site are DELETED (grep for all of them — listeners, catch blocks, guards that only set messages). `singleExportNote` becomes an info toast + log entry. The status bar's right zone gains a "Log" toggle button (`.btn`, text-xs) opening a simple panel (fixed, right side, scrollable, `--bg-1`, entries text-sm with time in `--text-3`... use `--text-2` for AA) listing the activity log newest-first; Escape closes it (guard: only when the brush isn't consuming Escape — check Viewer's handler scope; the panel toggle lives in App's window handler with isTypingTarget guard).
