# Background Job Queue Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Detect and heal become fire-and-forget: `d`/`h` enqueue a job and return, a background worker drains the queue, jumping between frames never interrupts or discards work, and a Heal-approved button bulk-enqueues the roll.

**Architecture:** One per-roll job queue (`jobs.rs`): a deque of `(kind, index)` jobs behind a mutex, drained by a single-flight worker task (the models serialize on their own mutexes anyway, so one worker loses nothing). Enqueueing the current frame pushes to the FRONT (the operator is looking at it); duplicates coalesce by `(kind, index)`. The worker resolves each job at run time from persisted state — the frame's sidecar threshold and strokes, the live model hashes — and lands results in the state cache; the cache is the source of truth, and the existing restore paths surface results into the UI whenever the frame is (or becomes) resident. For a frame that is resident when its job completes, the worker also updates the registry directly so the visible frame refreshes without a re-activation. In-flight frames are pinned against eviction. A roll swap or close clears the queue through the existing generation discipline. The scan and export queues are untouched (unification is explicitly out of scope).

**Tech Stack:** Existing stack; no new dependencies. Reuses `cache`, `masks::compose_heal_mask`, `with_inpainter_hashed`, `heal_with_progress`, and the ScanFlagGuard/generation patterns.

## Global Constraints

- Trunk-based: commit directly to `main`, atomic commits, tests green first. No emoji; no Co-Authored-By.
- The interactive commands stay responsive: `enqueue_job` returns immediately; nothing UI-visible blocks on model work. Every long-running state change is narrated by events with guaranteed terminal emission per job (`job-done` XOR `job-error`), plus a queue-level `queue-idle` when the worker drains.
- Job execution semantics MUST match the export queue's transient path exactly: threshold and strokes read from the sidecar at run time, provenance computed inside `with_inpainter_hashed`, cache read-before-compute, cache write after fresh compute. A heal produced by a job, by the viewer, or by an export must be indistinguishable.
- A job for a resident frame updates the registry through the validating setters (`set_probs_built`/`set_healed`) only; a stale or evicted id degrades to cache-only, never an error.
- Eviction pinning: the frame an active job is processing is never evicted mid-job (its result would be discarded). The pin covers exactly the active job's image id, not the whole queue.
- Generation discipline: the worker snapshots the roll generation at spawn and re-checks before each job; `RollState::open`/`close` clear the queue; a mid-job swap discards the result through the existing validating writes (sidecar via generation-checked methods, registry via id validation).
- Frontend keys: `d`/`h` enqueue (current frame to the front); the existing `detecting`/`healing` flags now mean "a job for the CURRENT frame is running" and drive the same status copy; `heal-progress` events keep working for the current frame.
- Filmstrip: queued and running frames get visually distinct markers (plain shapes/text, no emoji, contrast-checked like existing badges).
- Rust fmt + clippy `-D warnings`; svelte-check 0 errors (4-warning baseline); vitest green.

### Task 1: Job queue state

**Files:**
- Create: `app/src-tauri/src/jobs.rs`
- Modify: `app/src-tauri/src/lib.rs` (`mod jobs;`)
- Test: inline in `jobs.rs`

**Interfaces:**
- Produces:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum JobKind {
    Detect,
    Heal,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize)]
pub struct Job {
    pub kind: JobKind,
    pub index: usize,
}

/// Per-roll job queue. One worker drains it (single-flight `running` flag,
/// drop-guard cleared); `pinned` is the image id of the frame the active
/// job is processing, which eviction must skip.
#[derive(Default)]
pub struct JobQueue {
    queue: Mutex<VecDeque<Job>>,
    pub running: AtomicBool,
    pinned: Mutex<Option<u64>>,
}
```

Methods (all `&self`, all poison-safe via `map_err`):
- `enqueue(&self, job: Job, front: bool) -> Result<bool, String>` — false when an equal job is already queued (coalesced); `front` pushes ahead of everything (current-frame priority; an equal queued job is MOVED to the front rather than duplicated).
- `pop(&self) -> Result<Option<Job>, String>`.
- `clear(&self) -> Result<usize, String>` — returns how many were dropped (roll swap/close).
- `len(&self) -> Result<usize, String>` and `is_empty`.
- `pin(&self, id: Option<u64>) -> Result<(), String>` and `pinned(&self) -> Result<Option<u64>, String>`.
- `clear_running(&self)` — the drop-guard hook, mirroring `RollState::clear_scanning`.

### Task 2: The worker

**Files:**
- Modify: `app/src-tauri/src/lib.rs` (worker task + `enqueue_job` command + events + registration + manage state)

**Interfaces:**
- Consumes: everything the export queue's transient path uses, plus `JobQueue`, plus the registry-resident fast path pieces (`Images::{image, threshold_mask, set_probs_built, set_healed}`, `build_prob_pyramid`, `Pyramid::build`).
- Produces:
  - Managed `jobs::JobQueue` in the builder.
  - Command `enqueue_job(app, ..., kind: jobs::JobKind, index: usize, front: bool) -> Result<(), String>`: validates the index against the roll, enqueues (coalescing), emits `job-queued { index, kind }`, and starts the worker if not running (compare_exchange on `queue.running`; on setup failure clear the flag before returning the error — the scan_roll discipline).
  - The worker task (a free async fn spawned once per drain): snapshot `generation`; loop `queue.pop()`:
    1. Generation check (break on mismatch; the queue was cleared anyway).
    2. Resolve frame meta: path, file_name, threshold, strokes via `export_frame_meta(index)` (same one-lock read the export queue uses); resolve `roll.image_id(index)` for residency.
    3. Emit `job-started { index, kind }`.
    4. **Detect job**: if resident and the registry already has probs, skip (done). Else: resident → detect on the registry image Arc; not resident → `decode_stage`. Either way write the probs cache; if resident, also `build_prob_pyramid` + `set_probs_built`; then `record_scan_result(generation, index, Some(count), Some(bboxes))` and emit the existing `roll-progress` event shape so the filmstrip count/rings update exactly like a scan (reuse the `RollProgress` struct).
    5. **Heal job**: mirror the export transient body EXACTLY (validate strokes; probs from registry if resident else detect — reading through the probs cache first via `read_probs`, which the export path does not do today but SHOULD here to avoid re-detecting: check the cache before running the model; compose; `with_inpainter_hashed` with cache read/write inside as in `run_heal`) — with one addition: `queue.pin(Some(image_id))` around the resident-frame body (pin BEFORE reading registry Arcs, unpin in all exits — use a small drop guard so an early `?` cannot leave a stale pin). On success, if resident: build pyramid + `set_healed`; emit `heal-progress` events (existing struct) only when the job's frame is the pinned/resident one so the current-frame progress display works.
    6. Emit `job-done { index, kind }` or `job-error { index, kind, message }` per job; ALWAYS one of the two (wrap the per-job body in a fn returning Result, match at the call site — the export queue's shape).
    7. Loop until `pop()` returns None, then emit `queue-idle {}` and let the drop guard clear `running`.
  - `RollState::open`/`close` call sites in lib.rs additionally `queue.clear()` (the queue state is app-managed, accessible where open_roll/close_roll commands run).
  - `evict_over_budget` skips `queue.pinned()` ids alongside the current±1 window.

Event structs: `JobEvent { index: usize, kind: jobs::JobKind }` (used for queued/started/done), `JobError { index: usize, kind: jobs::JobKind, message: String }`.

### Task 3: Frontend enqueue semantics

**Files:**
- Modify: `app/src/App.svelte`
- Modify: `app/src/lib/Filmstrip.svelte`

**Interfaces:**
- Consumes: `enqueue_job`, events `job-queued`/`job-started`/`job-done`/`job-error`/`queue-idle`, existing `heal-progress` and `roll-progress` handling.
- Produces:
  - In roll mode, `requestDetect` and `requestHeal` become enqueues: `invoke("enqueue_job", { kind: "detect" | "heal", index: currentIndex, front: true })` and return — no more await-until-done, no `if (activating) return` guard needed for heal (the job resolves state at run time), though the heal path keeps the has-detection-or-queue-results guard for UX (enqueue detect first automatically when needed: enqueue detect THEN heal — the queue runs them in order; both front-inserted in that order means heal lands behind detect... enqueue heal with `front: true` FIRST, then detect with `front: true`, so detect ends up ahead. Do it in that order and comment why).
  - Single-image mode (no roll): `requestDetect`/`requestHeal` keep their current direct-invoke behavior (the queue is roll-only) — branch on `roll`.
  - Job state tracking: `jobStates: Record<number, "queued" | "running"> = $state({})` updated from the events (`job-done`/`job-error` delete the entry; `job-error` also surfaces the message via `error`). `detecting`/`healing` flags become derived-ish: set true when a `job-started` arrives for `currentIndex` with the matching kind, false on its done/error — preserving the existing status-line copy and the heal progress display (heal-progress events already carry the frame's image id; they keep working).
  - `queue-idle` clears any straggler entries.
  - Filmstrip: `jobStates` passed as a prop; a queued frame shows a small hollow marker, a running frame shows a filled one (distinct from the approved check and exported tag; plain characters, contrast-checked).
  - Heal-approved button in the header (roll mode, next to Export approved): enqueues a heal job (back of queue) for every approved frame; disabled when none approved. Plain label "Heal approved".

## Definition of done

- `d`/`h` return instantly; navigation never interrupts or discards model work; results are cache-durable and registry-live for resident frames.
- One worker, current-frame priority, coalesced duplicates, generation-safe against roll swaps, eviction-safe via pinning, terminal events per job.
- Heal-approved bulk enqueue exists; export becomes cache-hit-fast after it.
- NOT here: scan/export queue unification, job cancellation UI, cross-roll queue persistence.
